🛡️ UBS REST API

Version 3.0 | Universal Blocklist Syntax
Base URL: {{ base_url }}

📚 API Übersicht

Diese REST API ermöglicht das Parsen, Konvertieren, Validieren und Analysieren von UBS (Universal Blocklist Syntax) Dateien.

Features: Machine Learning Analyse, 21+ Exportformate, Echtzeit-Validierung, Statistiken & Analytics

Authentifizierung: HTTP Basic Auth (Username/Password erforderlich)

POST /api/parse Required First

Parse und validiere eine UBS-Datei. Dies ist der erste Schritt - die Datei wird im Server-Speicher geladen und steht für weitere Operationen zur Verfügung.

📤 Request Body

Content-Type: multipart/form-data
Field: "file" (UBS file)

💻 cURL Beispiel

curl -X POST {{ base_url }}/api/parse \ -u username:password \ -F "file=@blocklist.ubs"

📥 Response

{ "success": true, "rules_count": 1234, "metadata": { "title": "My Blocklist", "version": "1.0" }, "errors": [] }
GET /api/convert

Konvertiere die aktuell geladene UBS-Datei in ein beliebiges Zielformat. Die Datei wird als Download zurückgegeben.

🔧 Query Parameters

format=FORMAT_NAME (siehe verfügbare Formate unten)

💻 cURL Beispiel

curl -X GET "{{ base_url }}/api/convert?format=hosts" \ -u username:password \ -o blocklist.txt

📋 Verfügbare Formate (21+)

adblock
dnsmasq
unbound
bind

proxypac

littlesnitch
pgl

cloudflare

📥 Response

Content-Type: text/plain (Datei-Download)
Content-Disposition: attachment; filename="blocklist.FORMAT"
GET /api/validate

Führe eine umfassende Validierung der aktuell geladenen UBS-Datei durch. Prüft Syntax, Semantik und optionale DNS-Checks.

💻 cURL Beispiel

curl -X GET {{ base_url }}/api/validate \ -u username:password

📥 Response

{ "valid": true, "issues": [ { "severity": "warning", "line": 42, "message": "Duplicate rule detected", "suggestion": "Remove duplicate entry" } ], "stats": { "errors": 0, "warnings": 1, "info": 3 } }
POST /api/ml-analyze ML Powered

Machine Learning Analyse von Domains zur Erkennung von Malware, Phishing und verdächtigen Mustern. Verwendet trainierte ML-Modelle für Threat Intelligence.

📤 Request Body

{ "domains": [ "example.com", "suspicious-site.xyz", "malware-domain.net" ] }

💻 cURL Beispiel

curl -X POST {{ base_url }}/api/ml-analyze \ -u username:password \ -H "Content-Type: application/json" \ -d '{"domains": ["example.com", "test.xyz"]}'

📥 Response

{ "success": true, "count": 2, "analyses": [ { "domain": "example.com", "risk_score": 15.3, "threat_level": "LOW", "category": "legitimate", "confidence": 0.95, "recommendation": "ALLOW" } ] }
GET /api/stats

Detaillierte Statistiken über die aktuell geladene UBS-Datei. Zeigt Verteilung nach Regeltypen, Kategorien und Schweregrad.

💻 cURL Beispiel

curl -X GET {{ base_url }}/api/stats \ -u username:password

📥 Response

{ "total_rules": 1234, "by_type": { "domain": 1000, "url": 200, "regex": 34 }, "by_severity": { "critical": 45, "high": 234, "medium": 567 }, "unique_domains": 1150, "wildcard_count": 89 }
