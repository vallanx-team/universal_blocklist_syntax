Auto-Categorization

Automatically categorize domains into tracking, ads, malware, cdn, social, and other categories using machine learning.

from ubs_machine_learning import DomainCategorizer

categorizer = DomainCategorizer()

prediction = categorizer.predict_category("tracking-pixel.com")
print(f"Category: {prediction.predicted_category}")

Detect tracking and malware patterns using heuristics and signatures.

🔬 Python API Examples

Basic Usage

from ubs_parser import UBSParser, UBSConverter

# Parse UBS file
parser = UBSParser()
with open('rules.ubs', 'r') as f:
    parser.parse(f.read())

# Convert to hosts format
converter = UBSConverter(parser)
hosts = converter.to_hosts()
print(hosts)

Advanced ML Analysis

from ubs_machine_learning import AdvancedMLAnalyzer

# Create analyzer with trained models
analyzer = AdvancedMLAnalyzer(parser)

# Comprehensive analysis
result = analyzer.analyze_domain_comprehensive("suspicious.com")
print(f"Risk Score: {result['risk_score']:.1f}/100")
print(f"Threat Level: {result['threat_level']}")
print(f"Recommendation: {result['recommendation']}")

# Batch analysis and report
domains = ["domain1.com", "malware.tk", "tracker.net"]
analyses = analyzer.batch_analyze(domains)
analyzer.generate_report(analyses, "report.txt")
analyzer.export_suggested_rules(analyses, "rules.ubs")

💡 Complete Workflow Examples

ML-Enhanced Security Pipeline

#!/bin/bash
# Complete ML-enhanced security pipeline

# 1. Load production config
ubs-tool profile load production

# 2. Merge existing lists
ubs-tool merge corporate.ubs security.ubs malware.ubs --output base.ubs

# 3. Validate thoroughly

# 8. Final optimization
ubs-tool optimize enhanced.ubs --output final.ubs --aggressive

# 9. Generate analytics
ubs-tool analytics final.ubs --format all --charts

# 10. Convert to all formats
ubs-tool convert-all final.ubs --output ./production/

echo "✅ Pipeline Complete!"

⏱️ TTL Extension (Module 11)

What is the TTL Extension?

The ubs_ttl_extension.py module extends UBSConverter with per-rule TTL (Time To Live) support. It is a standalone file — ubs_parser.py stays completely untouched. Just swap UBSConverter for UBSConverterTTL and you gain full TTL-aware output.

UBS Syntax with TTL

# Short TTL for critical malware (1 minute)
malware.com :ttl=60 :severity=critical

# Standard TTL for trackers (5 minutes)
tracker.com :ttl=300 :category=tracker

# Long TTL for stable blocklists (1 hour)
stable-ad.com :ttl=3600

# Very long TTL for permanent blocks (24 hours)
spam.com :ttl=86400

# Combined with actions
bad-cdn.com :action=null :ttl=600

# Without TTL → uses default (300 s)
default-domain.com

Quick Start

from ubs_parser import UBSParser
from ubs_ttl_extension import UBSConverterTTL

parser = UBSParser()
parser.parse(open('rules.ubs').read())

converter = UBSConverterTTL(parser)

# TTL-specific formats
print(converter.to_unbound_ttl())
print(converter.to_dnsmasq_ttl())
print(converter.to_bind_ttl())
print(converter.to_pihole_ttl())
print(converter.to_coredns_ttl())
print(converter.to_ttl_report())

Convenience Function

from ubs_ttl_extension import convert_with_ttl

# Formats: unbound, bind, dnsmasq, pihole, coredns, report
result = convert_with_ttl(open('rules.ubs').read(), format='unbound')

Migration: 2 Lines to Change

Before After
from ubs_parser import UBSParser, UBSConverter from ubs_parser import UBSParser
from ubs_ttl_extension import UBSConverterTTL
converter = UBSConverter(parser) converter = UBSConverterTTL(parser)

All original methods (to_hosts(), to_adblock(), etc.) continue to work unchanged. to_unbound(), to_bind(), and to_dnsmasq() automatically use TTL when :ttl= modifiers are present.

Available Methods

Method Format Notes
to_unbound_ttl()Unboundlocal-data with TTL per record
to_bind_ttl()BINDZones grouped by TTL value
to_dnsmasq_ttl()dnsmasqaddress=/domain/ip#ttl
to_pihole_ttl()Pi-holeBlocklist + TTL recommendations
to_coredns_ttl()CoreDNSCorefile template with TTL
to_ttl_report()ReportDistribution, stats, recommendations

🐛 Troubleshooting

Common Issues

1. Configuration Not Loading

# Check config file locations
ubs-tool config-show

# Generate new config
ubs-tool config-init

# Specify config explicitly
ubs-tool --config my-config.yaml validate rules.ubs

2. Low ML Prediction Accuracy

Solution

Retrain with more diverse data. Ensure your training data covers all categories well.

# Check training data distribution
ubs-tool stats rules.ubs

# Add more examples
ubs-tool merge rules.ubs additional-examples.ubs --output better-training.ubs

3. Slow Performance

# Enable performance features
ubs-tool config-edit performance.use_bloom_filter true
ubs-tool config-edit performance.use_trie true

# Optimize rules
ubs-tool optimize rules.ubs --output optimized.ubs --aggressive

# Run benchmark
ubs-tool benchmark rules.ubs

# Detect false positives
ubs-tool check-false-positives rules.ubs

# Simulate traffic to test
ubs-tool simulate rules.ubs --requests 10000

# Test specific URLs
ubs-tool test rules.ubs --url https://suspected-false-positive.com

📝 UBS Syntax Reference

Basic Domain Blocking

# Simple domain
example.com

# Wildcard (all subdomains)
*.ads.example.com

# Regex pattern
~^ad[sx]?\d+\..*$

With Modifiers

# Block with logging
tracker.com :block :log

# Block only third-party
||analytics.com^ :third-party

# High severity with category
malware.com :severity=critical :category=malware

🎓 Best Practices

ML Model Training

  1. Use diverse training data - Include various categories
  2. Regular retraining - Retrain as you add new rules
  3. Validate predictions - Manually review high-confidence predictions
  4. Tune thresholds - Adjust based on your needs
  5. Combine with traditional rules - Use ML as augmentation

Confidence Thresholds

Range Confidence Action
≥ 0.9 Very High Auto-block recommended
0.7-0.9 High Review and likely block
0.5-0.7 Medium Manual review required
< 0.5 Low Investigate before action

Performance Optimization

  • Run optimize before deployment
  • Use DNS checks periodically, not always
  • Enable Bloom filter and Trie for large lists
  • Prefer simple domains over regex when possible
  • Use wildcards efficiently

Testing Workflow

  1. Validate syntax: ubs-tool validate rules.ubs
  2. Check false positives: ubs-tool check-false-positives rules.ubs
  3. Test URLs: ubs-tool test rules.ubs --batch critical-urls.txt
  4. Simulate traffic: ubs-tool simulate rules.ubs --requests 50000
  5. Run benchmarks: ubs-tool benchmark rules.ubs
